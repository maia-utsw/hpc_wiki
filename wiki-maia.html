The conda environment generally takes up a lot of space. You need to place the conda environment inside /data/maia/your_folder. The following is how to configure the conda environment.

Try to use the provided conda software and avoid installing a new conda or miniconda software yourself. Simply add the following to /home/Sxxx(ID)/.bashrc to use the conda.


# >>> conda initialize >>>
# !!! Contents within this block are managed by 'conda init' !!!
__conda_setup="$('/apps/anaconda3/bin/conda' 'shell.bash' 'hook' 2> /dev/null)"
if [ $? -eq 0 ]; then
    eval "$__conda_setup"
if [ $? -eq 0 ]; then eval "$__conda_setup
    if [ -f "/apps/anaconda3/etc/profile.d/conda.sh" ]; then
    . "/apps/anaconda3/etc/profile.d/conda.sh"
    else
    export PATH="/apps/anaconda3/bin:$PATH"
    fi
unset __conda_conda.sh
unset __conda_setup
# <<< conda initialize <<<

The next step is to configure the path to the conda environment, you need to open the following file (if you don't have it please create it yourself) /home/sxxxxx(ID)/.conda/environments.txt

Then enter the path to your environments in the txt. 

/data/maia/xxxx/conda_envs/xxx

Like my txt: 

/data/maia/yli/conda_envs/py38

If you want to move the current environments directly to /data/maia/your_folder, use the following command:

mv  /home/Sxxx(ID)/old_conda_path  /data/maia/xxxx/conda_envs/

Remember to configure the path in txt after you move it.

/home/Sxxx(ID)/.conda/pkgs
It is a cache file of pip, should be good to delete.
